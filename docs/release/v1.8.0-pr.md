# Fabric v1.8.0 — Stabilization & 3-Client Narrowing

**Branch**: `release/v1.8.0-stabilization` → `main`

## Summary

Combined deprecation + stabilization release. Narrows AI client surface from 6 to 3 (Claude / Codex / Cursor), hardens durability primitives (atomic writes, signal-safe shutdown, ledger queue), reframes doctor from "baseline promoter" to "consistency repairer", and ships the rule-sync orchestrator that 4 hot-path consumers (3 MCP tools + watcher) now depend on.

## What changed

### Foundation primitives (Layer 0/1)
- **Atomic write helper** (`@fenglimg/fabric-shared/node/atomic-write`) — tmp+rename + optional fsync; consumes 22 callsites across CLI + server
- **FabricError taxonomy** — 5 sub-trees (ConfigError / RuleError / IOFabricError / MCPError / InitError); `_error.ts` collapsed from 213→85 lines, single `instanceof FabricError` branch in normalizeApiError
- **Per-path ledger queue** — poison-resistant Promise-chain serialization; concurrent writes to same ledger path are ordered, failures don't poison the chain
- **rule-sync orchestrator** (`ensureRulesFresh` / `reconcileRules`) — single source of truth for rule freshness; 500 ms global cooldown debounce; watcher invalidates cooldown so file edits are picked up immediately; `reconcileRules` writes meta, `ensureRulesFresh` doesn't (hot-path safe)

### Stability hardening (Layer 2 P0)
- **3-client narrow** (Claude/Codex/Cursor); zod `.passthrough()` keeps legacy `clientPaths.*` keys parseable; doctor `legacy_client_path_present --fix` cleans them
- **Signal handlers** (SIGINT/SIGTERM/SIGHUP) — drain in-flight requests up to 5 s, fsync ledger, then `server.close()` then `process.exit(0)`. Double-signal forces immediate exit. Closes Claude Code #15945 zombie pattern.
- **Cross-process serve lockfile** — `.fabric/.serve.lock` PID file with `process.kill(pid, 0)` liveness check; ESRCH=stale (auto-recover), EPERM=conservative live, no error=live; `--force` override
- **Claude MCP config path fix** — was incorrectly writing to `.claude/settings.json` (hooks/permissions only per Claude Code spec); now `<projectRoot>/.mcp.json` (project) or `~/.claude.json` (user); deep-merge preserves OTHER `mcpServers` entries
- **`--reapply` data-loss fix** — events.jsonl bytes preserved; agents.meta.json preserved when `.fabric/rules/*.md` exists (protects AI-built rule tree); regenerates only when rules dir is empty
- **readEventLedger tail-tolerance** — trailing partial line surfaces as `LedgerWarning` (was silently dropped); doctor `event_ledger_partial_write --fix` truncates after backing up to `<ledger>.corrupted.{ts}`
- **Tool schemas + annotations** — schemas lifted to `@fenglimg/fabric-shared/schemas/api-contracts`; per-tool annotations (readOnlyHint / idempotentHint / destructiveHint / openWorldHint / title); golden contract snapshots
- **MCP entry wiring** — `ensureRulesFresh` runs at the top of all 3 tool handlers; `RuleSyncReport.warnings` flow through to `response.warnings` for AI consumers
- **Startup full reconcile** — server blocks on `reconcileRules` before accepting MCP requests; rules added offline are visible immediately on restart
- **Chokidar watcher → cache-invalidate-only** — extends to `.fabric/rules/`; no ledger writes from watcher, no race with concurrent doctor/serve
- **MCP payload guard** — 16 KB warn / 64 KB hard limit (`MCP_PAYLOAD_TOO_LARGE` 413); thresholds configurable via `fabric.config.json mcpPayloadLimits`
