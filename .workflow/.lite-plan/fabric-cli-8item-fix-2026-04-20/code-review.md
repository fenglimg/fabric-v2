**1. `packages/cli/src/commands/init.ts`**
- **Medium** - Cross-Platform Execution Bug - `packages/cli/src/commands/init.ts:~268`
  - **Issue:** `childProcess.execFileSync(manager, installArgs)` relies on `manager` being `"npm" | "pnpm" | "yarn"`. On Windows systems, invoking these package managers directly via `execFileSync` without `.cmd` wrapper or `shell: true` will throw `ENOENT`.
  - **Fix Suggestion:** Add `shell: process.platform === "win32"` to the `execFileSync` options to ensure safe Windows execution, or rely on `cross-spawn`.
- **Low** - Sub-optimal `mkdir` - `packages/cli/src/commands/init.ts:~196`
  - **Issue:** `mkdirSync(fabricDir, { recursive: false })` relies on the `prepareFreshPath` guarantee that the folder does not exist. However, if a race condition occurs, `EEXIST` will be thrown.
  - **Fix Suggestion:** Switch to `{ recursive: true }` as a standard defensive mechanism.

**2. `packages/cli/src/commands/bootstrap.ts`**
- **Low** - Codex Header Deduplication Edge Case - `packages/cli/src/commands/bootstrap.ts:~186`
  - **Issue:** In `stripExistingCodexBootstrap`, if `AGENTS.md` contains the `# Fabric Bootstrap` heading but does *not* start with it (e.g., placed at the bottom), `existing.startsWith("# Fabric Bootstrap")` evaluates to false, returning the file unmodified.   - **Fix Suggestion:** This proves Codex's claim of conservative, non-destructive editing. To make it cleaner, loosen the `startsWith` constraint to allow slicing out the existing header block anywhere in the file.
